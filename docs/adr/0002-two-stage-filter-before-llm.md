# 一次フィルタ（ルールゲート）と LLM 分析の二段階

すべての Detected Change を Cursor SDK に送るのではなく、二段階にする。(1) **ルールゲート（一次フィルタ）** で明らかなノイズのみ LLM 対象外とする。高重み公式ソース（厚労省療養費・消費者庁措置命令等）またはキーワード一致は無条件で LLM へ。低重みかつキーワード0件の一般ニュース系は Analysis を作らない。(2) **Cursor SDK** はゲート通過分に対し、重要度・対象業態・実務影響・関連度（`relevance: low` 含む）を付与する。技術スタックの「関連度判定」はゲート後の意味判断とする。

取得失敗の Change はゲート対象外（別節・別カウンタ）。差分なしは Change 自体を作らない（従来どおり）。

**Status**: accepted

**Considered Options**: 一次フィルタのみでハード除外（A）/ 差分あれば全件 LLM（B）

**Consequences**: `packages/core` にゲート判定と Analysis 必須条件を明示。日次レポート本文は Analysis ありを重要度順。末尾に「参考・未分析」節を設け、ゲート不通過の Change をタイトル・原典URL・ゲート理由のみ掲載（LLM 要約なし）。F1-3「優先抽出」は「高重みは無条件 LLM」と読み替える。
